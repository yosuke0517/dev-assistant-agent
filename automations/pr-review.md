# Cursor Automation: PR Review (Assign PR reviewers)

dev-assistant-agentが作成したPRを自動レビューするためのCursor Automationsテンプレート。

## トリガー

- **タイプ**: GitHub - PR open / push
- **条件**: PRが作成またはプッシュされた時

## 設定

```yaml
trigger: github.pull_request.opened, github.push
tools:
  - Open pull request
  - Send to Slack
mcp:
  - GitHub MCP
  - Slack MCP
```

## プロンプト

```
あなたはコードレビューアーです。以下のPRをレビューしてください。

## 手順

1. PRの変更内容（diff）を確認してください
2. 関連するIssueがあればその仕様も確認してください
3. 以下の観点でレビューしてください：

### レビュー観点
1. **仕様充足性**: Issue仕様の要件が全て実装されているか
2. **スコープ逸脱**: 仕様に含まれていない余分な変更がないか
3. **ロジックバグ**: ロジックバグ・エッジケースの見落とし
4. **セキュリティ**: XSS、SQLインジェクション、コマンドインジェクション等
5. **パフォーマンス**: パフォーマンス上の問題
6. **リファクタリング**: コードの可読性・保守性の改善

4. リスクを5段階で評価してください：
   - 低リスク（1-2）: 自動承認を提案
   - 中リスク（3）: レビュー推奨
   - 高リスク（4-5）: 必須レビュー & 適切なレビュアーをアサイン

5. レビュー結果をPRコメントとして投稿してください
```

## 備考

- Cursor Bugbot と併用すると二重チェック体制になる
- 優先度: ★★★（最優先で導入推奨）
