# Cursor Automation: Summarize Changes Daily / Weekly

リポジトリの変更を定期的にサマリーとしてSlackに投稿するCursor Automationsテンプレート。
ドメイン情報の更新にもカスタマイズ可能。

## トリガー

- **タイプ**: Timer (cron)
- **日次**: `0 9 * * 1-5` (平日 9:00 UTC)
- **週次（ドメイン情報更新用）**: `0 9 * * 1` (毎週月曜 9:00 UTC)

## 設定

```yaml
trigger: timer.cron("0 9 * * 1-5")  # 日次
# trigger: timer.cron("0 9 * * 1")  # 週次（ドメイン情報更新用）
tools:
  - Open pull request  # ドメイン情報更新PR作成用
  - Send to Slack
  - Memories           # 過去のパターン学習
mcp:
  - GitHub MCP
  - Slack MCP
```

## プロンプト（日次サマリー）

```
前日にマージされたPRを分析し、変更サマリーをSlackに投稿してください。

## 手順

1. 直近24時間（または前営業日以降）にマージされたPRを取得
2. 各PRの変更内容をカテゴリ分類：
   - 🚀 新機能
   - 🐛 バグ修正
   - ♻️ リファクタリング
   - 📝 ドキュメント
   - 🔧 インフラ・CI/CD
3. 以下の形式でSlack #dev-updates に投稿：

📊 *Daily Changelog - [日付]*

🚀 *新機能*
- PR #XX: [タイトル] (@author)

🐛 *バグ修正*
- PR #YY: [タイトル] (@author)

合計: X件のPRがマージされました
```

## プロンプト（週次ドメイン情報更新）

```
直近1週間のマージ済みPRを分析し、ドメイン情報の更新を行ってください。

## 手順

1. 直近1週間にマージされたPRを取得
2. 以下の変更を特定：
   - ドメインモデルの変更（エンティティ、値オブジェクト）
   - API変更（エンドポイントの追加・変更・削除）
   - DBスキーマ変更
   - ビジネスルールの変更
3. docs/domain-info.md を更新するPRを作成
4. 変更サマリーをSlack #dev-updates に投稿
```

## 備考

- Memories ツールを有効化すると過去のパターンを学習して改善される
- マルチリポジトリの場合は各リポジトリに個別設定が必要
- 優先度: ★★★（チーム向け情報共有・ドメイン知識の蓄積）
